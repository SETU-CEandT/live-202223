Python Programming

Introduction

We are going to learn a language and we need to learn it by yesterday. How should we approach this? The strategy that I adopt is as follows:

  • Get a set of programming tasks that are easy to do in my programming language of choice.

    These tasks should be well within my current capability since the aim of the exercise is to learn a programming language, and not to learn how to program.

    The following pages contain one such list that we will use for our first assignment.

  • As I implement each task I build up a reference sheet of programming idioms in the new language.

    And as I'm lazy, I find and use suitable pre-existing reference sheets . . .

    • Python Overview

      • Lists
      • Dictionaries
      • Looping
      • Functions
      • Files and Exceptions

    • Matplotlib

    Each sheet is only two pages - print them out and stick them to your bedroom ceiling so it is the first thing you see every morning!

  • Next, and this is the important step, I try to change my thinking process/coding structures to match the new language. I don't want to just write Java-like code in Python. Doing this, means I don't get any of the advantages that Python offers me over Java. See for example:

    • Six programming paradigms that will change how you think about coding
